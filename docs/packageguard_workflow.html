PackageGuard

A Two-Tier, Post-Generation Defense Against LLM Package Hallucinations (“Slopsquatting”)

1Intercept

Capture LLM output or a pip install request before any code runs.

2Extract Names

Pull package names from pip lines, imports & prose; map import→dist.

3Trust Engine

8 independent collectors gather live evidence for every package.

4Score & Verdict

Weighted average → PASS / WARN / BLOCK; non-existent = hard BLOCK.

5Enforce Policy

Tier 1 annotate/block/refine · Tier 2 proceed/prompt/refuse.

6Safe Output

Trustworthy answer, or a blocked install — supply chain protected.

1Intercept
Two entry points:
  • Tier 1 — OpenAI-compatible proxy in front of the LLM
  • Tier 2 — safeinstall CLI wrapping pip
Post-generation: acts on what the model actually said.
2Extract Names
  • Parse pip install lines (verbatim)
  • Parse import statements
  • Scan prose for package mentions
  • Filter stdlib + stopwords; PEP-503 normalise
  • import→dist aliases (e.g. paho→paho-mqtt)

4Score & Verdict
  • final = Σ(score·weight) / Σweight
  • ≥70 → PASS
  • 40–69 → WARN
  • <40 → BLOCK
  • Doesn’t exist → hard BLOCK
  • Veto signals abstain when clean; only vote on real evidence.
5Enforce
Tier 1 proxy:
  • annotate
  • block
  • refine (model self-corrects)
Tier 2 CLI:
  • PASS → install
  • WARN → prompt
  • BLOCK → refuse (exit 2)
Two Enforcement Surfaces
🛡️ Tier 1 — LLM Proxy (pre-emptive)
Sits between the user and the model as a drop-in OpenAI endpoint. Every completion is scanned before it reaches the developer.
annotate tag risky packages inline block strip unsafe install lines refine ask the model to fix its own answer
User → Proxy + Guard → LLM → scan + policy → safe answer
📦 Tier 2 — safeinstall CLI (last line)
Wraps pip so a hallucinated name can’t be installed even if it slips past the model — the trust check runs before pip executes any code.
PASS install proceeds WARN ask the user BLOCK refuse, exit code 2
safeinstall pip install X → trust engine → verdict → pip / stop

Verdict Legend

PASS ≥ 70Established, well-used, low-risk package
WARN 40–69Plausible but thin signal — user is warned, not blocked
BLOCK < 40Hallucinated / fresh squat / unsafe — refused

How this defeats slopsquatting

A fresh squat exists on the index, so a static allow-list waves it through.
Multi-signal trust catches low age, 0 downloads, no real repo, no usage.
Install-time scan inspects setup.py without executing it.
Refine loop makes the model remove the bad package itself.
★
Key Takeaway: Existence is not trust. A package can be real on a live index and still be a hallucinated squat. PackageGuard replaces a static allow-list with a live, multi-signal trust engine enforced at two points — the LLM proxy and the install command — so unsafe packages are caught after generation and before execution.
